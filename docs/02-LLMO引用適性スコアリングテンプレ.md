# LLMO引用適性スコアリングテンプレ

## 使用方法
1. 記事の各項目を確認し、0-20点で採点
2. 5項目の合計スコア（0-100点）を算出
3. 90点以上を目指し、不足項目を改善

## スコア判定

- **90-100点**：LLMO引用適性が非常に高い ✅
- **70-89点**：LLMO引用適性が高い。改善の余地あり 🔄
- **50-69点**：LLMO引用適性が中程度。大幅な改善が必要 ⚠️
- **0-49点**：LLMO引用適性が低い。構造の見直しが必要 ❌

## 改善テンプレート

### 1. 冒頭定義文のテンプレート

```
【キーワード】とは、【簡潔な定義】のことです。
具体的には、【具体例1】や【具体例2】などが該当します。
中小企業においては、【ターゲットの課題】を解決するために重要な概念です。
```

**例：**
```
AI導入とは、業務プロセスに人工知能技術を組み込むことです。
具体的には、顧客対応の自動化やデータ分析の効率化などが該当します。
中小企業においては、人手不足や業務効率化の課題を解決するために重要な取り組みです。
```

### 2. 箇条書き・番号付き手順のテンプレート

```
【手順名】の手順は以下の通りです：

1. 【ステップ1の具体的な内容】
2. 【ステップ2の具体的な内容】
3. 【ステップ3の具体的な内容】

各ステップの注意点：
・【注意点1】
・【注意点2】
・【注意点3】
```

### 3. チェックリストのテンプレート

```
【目的】のチェックリスト

□ 【チェック項目1】
□ 【チェック項目2】
□ 【チェック項目3】
□ 【チェック項目4】
□ 【チェック項目5】

【合計】項目中、【推奨数】項目以上を満たすことが推奨されます。
```

### 4. 数値・条件の明示テンプレート

```
【項目名】の条件：
・【条件1】：【具体的な数値・基準】
・【条件2】：【具体的な数値・基準】
・【条件3】：【具体的な数値・基準】

注意点：
・【注意点1】：【具体的な数値・基準】
・【注意点2】：【具体的な数値・基準】
```

### 5. 実務レベルの具体例テンプレート

```
【ケース名】の具体例：

【状況】：【具体的な状況説明】
【課題】：【具体的な課題】
【対応】：【具体的な対応方法】
【結果】：【具体的な結果・数値】

この事例から分かるポイント：
1. 【ポイント1】
2. 【ポイント2】
```

## 使用例

### 例：記事「AI導入 中小企業 失敗 防ぐ方法」

| 評価項目 | スコア | 評価内容 |
|----------|--------|----------|
| 1. 冒頭の定義文 | 15点 | 「AI導入の失敗とは」の定義があるが、やや長い（150文字） |
| 2. 箇条書き・番号付き手順 | 20点 | 失敗パターン5つ、対策手順3ステップを箇条書きで明示 |
| 3. テンプレ・チェックリスト | 15点 | 記事内に「失敗チェックリスト」を記載。ダウンロード版はなし |
| 4. 数値・条件・注意点 | 18点 | 導入費用の目安、失敗率の数値、注意点を5つ明示 |
| 5. 実務レベル | 20点 | 実際の失敗事例3つ、対策の具体的手順を記載 |

**合計スコア：88点**

### 改善提案（90点以上を目指す）

1. **不足項目：** 冒頭の定義文が長い
   **改善内容：** 定義文を100文字以内に短縮。「AI導入の失敗とは、準備不足や目的不明確により期待した効果が得られない状態のことです。」
   **想定工数：** 10分

2. **不足項目：** テンプレがダウンロード不可
   **改善内容：** チェックリストをPDF化し、ダウンロード可能にする
   **想定工数：** 30分

3. **不足項目：** 数値がやや少ない
   **改善内容：** 失敗率の具体的な数値（例：「中小企業のAI導入失敗率は約70%」）を追加
   **想定工数：** 15分

**改善後の想定スコア：93点** ✅
